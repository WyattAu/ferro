### RUSTSEC-2023-0071 (rsa crate)

**Severity:** 5.9/10 (Medium)  
**Status:** No upstream fix available. The `rsa` crate is a transitive dependency of `jsonwebtoken` (OIDC authentication).  
**Mitigation:** The vulnerable code path is only reachable when OIDC is configured (`--oidc-*` flags). For non-OIDC deployments, this code is dead.  
**Documented in:** `SECURITY.md`

## Known Limitations

1. **ActivityPub delivery is fire-and-forget** — no retry queue for failed deliveries
2. **HTTP signature verification is available but not enforced** — inbox accepts unsigned activities
3. **WebSocket is notification-only** — clients cannot send commands over WebSocket
4. **Search index failures are silently logged** — by design (search should not block file ops)
5. **File versioning is in-memory metadata + disk content** — metadata lost on restart
6. **OIDC PKCE sessions are in-memory** — lost on restart (acceptable for single-instance)
7. **CalDAV/CardDAV stores are in-memory** — no persistent backend yet
8. **Tauri desktop is scaffold only** — system tray works, full GUI not implemented
9. **RUSTSEC-2023-0071** — unpatchable rsa crate vulnerability (OIDC-only code path)

## Conclusion

Ferro is a well-structured, safe, and performant Rust storage orchestrator. The codebase demonstrates:
- **Zero unsafe code** in any context
- **Zero clippy warnings** across all feature combinations
- **497 passing tests** with 0 failures
- **Sub-millisecond** metadata operations
- **Sub-6ms** file operations for 1MB files

The identified limitations are documented and represent future roadmap items, not blocking issues for deployment.
