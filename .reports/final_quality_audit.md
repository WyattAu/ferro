## Priority Remediation Order

### Must Fix (High Severity)

1. **Password hashing** (`crates/server/src/users.rs:148-153`): Replace SHA-256 with argon2/bcrypt. The current scheme has no key stretching and a static salt — trivially crackable with rainbow tables.
2. **Timing-unsafe password comparison** (`crates/server/src/users.rs:156-158`, `crates/server/src/simple_auth.rs:65`): Use `subtle::ConstantTimeEq` (already a dependency).
3. **Default WOPI token secret** (`crates/server/src/lib.rs:137`): Reject WOPI requests entirely when using the default secret, rather than just warning.
4. **Unbounded audit log** (`crates/server/src/audit.rs:28`): Implement ring buffer or max-entries limit.
5. **Unbounded recently_processed set** (`crates/server/src/lib.rs:138`): Add TTL-based eviction.
6. **Unbounded trash** (`crates/server/src/lib.rs:144`): Add max entries or periodic cleanup.

### Should Fix (Medium Severity)

7. **Unbounded share links** — Add expired share cleanup.
8. **Unbounded webhook configs** — Add max webhook count.
9. **Rate limiter cleanup never called** — Spawn periodic cleanup task.
10. **PKCE sessions never cleaned** — Add TTL-based cleanup.
11. **Duplicate AuditEntry/AuditLog in snapshots.rs** — Remove dead duplicate code.
12. **Share download loads entire file into memory** — Stream large files.
13. **Production `.unwrap()` calls in handlers** — Replace with proper error handling or `.expect()` with messages.

### Nice to Have (Low/Info)

14. Add documentation to all `pub fn`, `pub struct`, `pub trait` items.
15. Consider sharded rate limiter for better throughput.
16. Audit all `#[allow(dead_code)]` annotations for ongoing relevance.
