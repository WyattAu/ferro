# SOC 2 Audit Checklist

## Pre-Audit (30 days before)

### Documentation
- [ ] All policies documented and reviewed
- [ ] Risk assessment completed
- [ ] Vendor assessments completed
- [ ] Business continuity plan documented
- [ ] Incident response plan documented

### Technical Controls
- [ ] MFA enabled for all users
- [ ] RBAC implemented and reviewed
- [ ] Encryption at rest enabled
- [ ] Encryption in transit enabled
- [ ] Logging enabled and monitored
- [ ] Vulnerability scanning automated
- [ ] Patch management process defined

### Evidence Collection
- [ ] Access logs collected
- [ ] Change logs collected
- [ ] Security scan results collected
- [ ] Test coverage reports collected
- [ ] Performance metrics collected

## Audit Week

### Day 1: Opening Meeting
- [ ] Present organization overview
- [ ] Present control environment
- [ ] Provide documentation access

### Day 2-3: IT Controls
- [ ] Provide access control evidence
- [ ] Provide change management evidence
- [ ] Provide monitoring evidence

### Day 4: Security Controls
- [ ] Provide security scan results
- [ ] Provide vulnerability management evidence
- [ ] Provide incident response evidence

### Day 5: Closing Meeting
- [ ] Review findings
- [ ] Discuss remediation plan
- [ ] Set timeline for responses

## Post-Audit (30 days after)

### Remediation
- [ ] Address all findings
- [ ] Document remediation actions
- [ ] Provide evidence of remediation

### Follow-up
- [ ] Respond to auditor questions
- [ ] Provide additional evidence if requested
- [ ] Sign management representation letter
