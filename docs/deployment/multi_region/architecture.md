## Failover Scenarios

### Single Region Failure

1. Health checks fail for 3 consecutive intervals (90s)
2. Load balancer removes region from active pool
3. Traffic rerouted to nearest healthy region
4. Database promoted in secondary region (if primary failed)
5. Storage accessed from regional replica
6. Alert sent to on-call team

### Multiple Region Failure

1. Load balancer reduces to remaining healthy regions
2. Critical services prioritized (auth, calendar sync)
3. Degraded mode activated (read-only for non-critical)
4. Remaining regions scale up to handle increased load
5. Recovery procedures initiated

### Database Primary Failure

1. Patroni/pg_auto_failover detects primary failure
2. Most up-to-date replica promoted to primary
3. Application servers reconnect (PgBouncer handles this)
4. Old primary repaired and rejoins as replica
5. Replication lag verified before accepting writes
