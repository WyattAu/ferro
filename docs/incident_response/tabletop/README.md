# Incident Response Tabletop Exercise

## Overview

This document describes the tabletop exercise process for incident response readiness.

## Objectives

1. Test incident response procedures
2. Identify gaps in communication
3. Validate escalation paths
4. Practice decision-making under pressure

## Participants

| Role | Name | Responsibility |
|------|------|----------------|
| Incident Commander | [Name] | Overall coordination |
| Technical Lead | [Name] | Technical response |
| Communications Lead | [Name] | Internal/external comms |
| Executive Sponsor | [Name] | Decision-making |

## Schedule

| Time | Activity |
|------|----------|
| 0:00 | Scenario introduction |
| 0:05 | Initial response |
| 0:15 | Triage and assessment |
| 0:30 | Mitigation actions |
| 0:45 | Recovery planning |
| 0:55 | Debrief and lessons learned |

## Scenarios

1. Database Outage - Complete database failure
2. Security Breach - Unauthorized access detected
3. Data Loss - Accidental data deletion

## Success Criteria

- All participants understand their roles
- Communication channels work effectively
- Escalation procedures are followed
- Recovery plan is clear and actionable
