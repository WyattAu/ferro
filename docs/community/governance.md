# Ferro Governance

## Overview

Ferro is governed by a community-driven process with clear roles and responsibilities.

## Decision Making

### Consensus-Based
- Discussion on GitHub/Discord
- RFC for major changes
- Maintainer approval for merging

### RFC Process
1. Create RFC document
2. Community discussion (2 weeks)
3. Maintainer review
4. Decision announcement
5. Implementation

### Voting
- Maintainers have voting rights
- Simple majority for most decisions
- Supermajority for breaking changes
- Unanimous for governance changes

## Roles

### Contributors
- Submit issues and PRs
- Participate in discussions
- Help with documentation
- Report bugs

### Maintainers
- Review and merge PRs
- Manage releases
- Mentor contributors
- Guide architecture

### Steering Committee
- Set direction
- Resolve conflicts
- Manage governance
- Represent community

## Responsibilities

### Contributors
- Follow code of conduct
- Follow contribution guidelines
- Be respectful and constructive

### Maintainers
- Review PRs in timely manner
- Provide helpful feedback
- Mentor contributors
- Maintain quality

### Steering Committee
- Set strategic direction
- Resolve disputes
- Manage resources
- Represent community

## Elections

### Maintainers
- Nominated by existing maintainers
- Voted by maintainers
- Term: 1 year
- Re-election: Yes

### Steering Committee
- Nominated by maintainers
- Voted by maintainers
- Term: 2 years
- Re-election: Yes

## Changes

### Governance Changes
- Require supermajority
- 30-day discussion period
- Public vote
- Announcement

### Role Changes
- Require maintainer approval
- 14-day discussion period
- Private vote
- Announcement

## Conflict Resolution

### Process
1. Discussion on GitHub/Discord
2. Mediation by maintainer
3. Escalation to steering committee
4. Final decision

### Principles
- Respectful discussion
- Focus on technical merits
- Community benefit
- Transparency

## Transparency

### Public
- All discussions
- All decisions
- All votes
- All finances

### Private
- Personal information
- Security vulnerabilities
- Legal matters

## Communication

### Channels
- GitHub (issues, PRs, discussions)
- Discord (real-time)
- Twitter/X (announcements)
- Blog (detailed updates)

### Guidelines
- Be respectful
- Be inclusive
- Be constructive
- Be professional
